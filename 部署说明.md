# 部署说明

服务器：使用 `DEPLOY_SERVER` 环境变量（默认 `root@120.26.0.177`，见 `.env.deploy.example`）  
游戏地址：http://主机:3000（主机从 DEPLOY_SERVER 解析，如 root@120.26.0.177 → 120.26.0.177）

---

## 一、四个入口（双击运行）

| 文件 | 用途 | 使用场景 |
|------|------|----------|
| **首次部署.bat** | 新服务器环境准备 | 换新服务器、或服务器重装后，安装 Node、MongoDB 等 |
| **快速部署.bat** | dist + client + package.json，服务器端 npm install | **日常更新（推荐）**，本地 Windows → 服务器 Linux |
| **上传部署.bat** | 全量部署（含 node_modules） | 仅当本地与服务器同系统时使用 |
| **服务器检查.bat** | 查看进程、端口、日志 | 服务崩了时先诊断 |
| **服务器重启.bat** | 仅重启服务，不恢复 | 进程挂了时快速拉起 |
| **回滚.bat** | 恢复到上一版部署 | 更新后出问题，从备份恢复 |
| **从服务器恢复.bat** | 从服务器下载生产包到本地 | 本地改崩时，获取当前生产环境内容 |

---

## 二、使用流程

### 1. 首次部署（新服务器）

1. 确保能 SSH 连接：`ssh %DEPLOY_SERVER%`（或 `ssh $env:DEPLOY_SERVER`，未设置时默认 root@120.26.0.177）
2. 双击 **首次部署.bat**，安装 Node、MongoDB、Swap 等环境（约 5 分钟）
3. 完成后，双击 **快速部署.bat**，完成首次代码部署（服务器端 npm install 安装依赖）

> 生产环境**不使用 Git**，仅部署 dist、client、package.json 等，依赖在服务器安装。

### 2. 日常更新（dist 部署）

**推荐：快速部署.bat**（本地 Windows → 服务器 Linux，依赖在服务器安装）

| 步骤 | 说明 |
|------|------|
| 1 | 双击 **快速部署.bat** |
| 2 | 本地：清理 dist → `npm run build` → 打包 dist + client + package.json + package-lock.json + init-mongodb.js |
| 3 | 上传到服务器 → 解压 → `npm install --production` → `node init-mongodb.js` → PM2 重启 |

**适用**：日常业务代码更新（server/src/*.ts、client/*、package.json 依赖变更）

**功能型道具**：在 `text-rpg-game/server/init-mongodb.js` 中新增或修改后，直接运行 **快速部署.bat** 即可。部署会打包并上传 `init-mongodb.js`，服务器解压后自动执行 `node init-mongodb.js`，新道具会插入/更新到数据库。

**备选：上传部署.bat**（全量部署，含 node_modules）

- 注意：本地是 Windows、服务器是 Linux 时，**不要用全量部署**（bcrypt 等原生模块不兼容）
- 仅当本地与服务器同系统时使用，或首次部署后改用快速部署

> 无需 `git push`，本地构建后直接上传。

### 3. 回滚（出问题时）

1. 双击 **回滚.bat**
2. 输入 `y` 确认
3. 从备份恢复上一版代码和数据库，并重启

> 回滚基于部署时的备份（快速部署和全量部署都会创建备份），不依赖 Git。备份保留最近 5 份。

### 4. 从服务器恢复（本地改崩时）

1. 双击 **从服务器恢复.bat**
2. 输入服务器密码
3. 将 `/opt/guaji` 下载到项目同级目录的 `guaji-从服务器恢复` 文件夹

> 下载内容含 text-rpg-game（dist、node_modules、client 等），不含 TypeScript 源码。可将该文件夹中的文件复制到当前项目覆盖。

## 三、脚本目录

实际脚本在 `scripts/` 下：

- `scripts/first-deploy.sh`：首次部署（仅环境）
- `scripts/upload-deploy-quick.ps1`：快速部署（推荐）
- `scripts/deploy-remote-quick.sh`：服务器端快速部署逻辑
- `scripts/upload-deploy.ps1`：全量上传部署
- `scripts/deploy-remote.sh`：服务器端全量部署逻辑
- `scripts/rollback.sh`：回滚逻辑
- `scripts/fix-rate-limit-on-server.ps1`：应急修复（仅上传 rate-limit.js）

**从服务器恢复** 无需 scripts。

## 四、生产环境规范

| 规范 | 说明 |
|------|------|
| **仅部署生产包** | 生产服务器不克隆 Git，只接收 dist、node_modules、client、package.json、init-mongodb.js |
| **无源码与测试** | 不包含 src/、scripts/test-*.js、*.test.ts、docs/ 等 |
| **仅运行时依赖** | node_modules 经 `npm prune --production` 精简，不含 Jest、ESLint、TypeScript |
| **部署前自动备份** | 每次部署前备份当前版本到 `/opt/guaji/backups/`，保留最近 5 份 |

## 五、登录与多端

- **多端踢下线**：同一账号在新设备/浏览器登录时，旧连接会被踢下线
- **IP 锁**：已暂时关闭（测试阶段），后续可按需启用
- **Nginx**：若游戏在 Nginx 后，需配置 `proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;` 以正确获取客户端 IP

---

## 六、常见问题

| 现象 | 处理 |
|------|------|
| 快速部署后报 `Cannot find module 'xxx'` | 服务器 package.json 可能未更新，手动上传：`scp text-rpg-game/server/package.json root@服务器:/opt/guaji/text-rpg-game/server/`，再 SSH 执行 `cd /opt/guaji/text-rpg-game/server && rm -rf node_modules && npm install --production && pm2 restart text-rpg-game` |
| 服务启动后立即崩溃 | 用 **服务器检查.bat** 查看 `pm2 logs text-rpg-game`，根据报错排查 |

---

## 七、修改服务器地址

若更换服务器，设置环境变量 `DEPLOY_SERVER` 即可（格式：`root@主机`）：

- **Windows CMD**：`set DEPLOY_SERVER=root@你的服务器IP`（或新建 `.env.deploy` 后加载）
- **PowerShell**：`$env:DEPLOY_SERVER = "root@你的服务器IP"`
- 未设置时默认使用 `root@120.26.0.177`
- 详见项目根目录 `.env.deploy.example`

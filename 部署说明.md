# 部署说明

服务器：**120.26.0.177**  
游戏地址：http://120.26.0.177:3000

---

## 一、四个入口（双击运行）

| 文件 | 用途 | 使用场景 |
|------|------|----------|
| **首次部署.bat** | 新服务器完整部署 | 换新服务器、或服务器重装后第一次部署 |
| **日常更新.bat** | 拉取代码并重新部署 | 改完代码并 `git push` 后，想更新到服务器 |
| **上传部署.bat** | 本地构建并上传 | 有本地修改但不想 push，直接部署当前代码 |
| **回滚.bat** | 恢复到部署前状态 | 更新后出问题，需要回退 |
| **从服务器恢复.bat** | 从服务器下载代码到本地 | 本地改崩了，用生产环境代码恢复 |

---

## 二、使用流程

### 1. 首次部署（新服务器）

1. 确保能 SSH 连接：`ssh root@120.26.0.177`
2. 双击 **首次部署.bat**
3. 按提示输入服务器 root 密码
4. 等待约 5–10 分钟（安装 Node、MongoDB、克隆代码、构建、启动）

### 2. 日常更新（推荐，需服务器有 Git）

1. 本地改完代码后：`git add .` → `git commit` → `git push`
2. 双击 **日常更新.bat**
3. 输入服务器密码，等待构建和重启完成

> 前提：服务器需用 **首次部署** 部署过（有 Git 仓库）。若之前用 上传部署，请继续用 上传部署 做更新。

### 3. 上传部署（免 push）

1. 本地有未 push 的修改
2. 双击 **上传部署.bat**
3. 本地构建后打包上传到服务器并重启

> 适合：之前用 上传部署 初始化过的服务器，或不想 push 时使用。

### 4. 回滚（出问题时）

1. 双击 **回滚.bat**
2. 输入 `y` 确认
3. 恢复到上一次部署前的代码和数据库状态

### 5. 从服务器恢复（本地改崩时）

1. 双击 **从服务器恢复.bat**
2. 输入服务器密码
3. 代码会下载到项目同级目录的 `guaji-从服务器恢复` 文件夹
4. 将该文件夹中的内容复制到当前项目覆盖，即可恢复

---

## 三、脚本目录

实际脚本在 `scripts/` 下：

- `scripts/first-deploy.sh`：首次部署逻辑
- `scripts/daily-update.sh`：日常更新逻辑
- `scripts/upload-deploy.ps1`：上传部署逻辑
- `scripts/rollback.sh`：回滚逻辑

**从服务器恢复** 无需 scripts，直接执行根目录的 `从服务器恢复.bat` 即可。

一般只需双击根目录的 `.bat` 文件即可。

---

## 四、修改服务器地址

若更换服务器，需修改以下文件中的 `120.26.0.177`：

- `scripts/首次部署.sh`（注释中的说明）
- `scripts/上传部署.ps1`（`$SERVER` 变量）
- 各 `.bat` 文件中的 `root@120.26.0.177`

# 部署说明

服务器：**120.26.0.177**  
游戏地址：http://120.26.0.177:3000

---

## 一、四个入口（双击运行）

| 文件 | 用途 | 使用场景 |
|------|------|----------|
| **首次部署.bat** | 新服务器环境准备 | 换新服务器、或服务器重装后，安装 Node、MongoDB 等 |
| **上传部署.bat** | 本地构建并上传部署 | 首次部署代码、或日常更新，改完代码后执行 |
| **回滚.bat** | 恢复到上一版部署 | 更新后出问题，从备份恢复 |
| **从服务器恢复.bat** | 从服务器下载生产包到本地 | 本地改崩时，获取当前生产环境内容 |

---

## 二、使用流程

### 1. 首次部署（新服务器）

1. 确保能 SSH 连接：`ssh root@120.26.0.177`
2. 双击 **首次部署.bat**，安装 Node、MongoDB、Swap 等环境（约 5 分钟）
3. 完成后，双击 **上传部署.bat**，完成首次代码部署

> 生产环境**不使用 Git**，仅部署 dist、node_modules、client 等运行所需文件。

### 2. 上传部署（日常更新）

1. 本地改完代码
2. 双击 **上传部署.bat**
3. 本地构建 → 打包生产包 → 上传到服务器 → 自动备份当前版本 → 解压 → 重启

> 无需 `git push`，本地构建后直接上传。

### 3. 回滚（出问题时）

1. 双击 **回滚.bat**
2. 输入 `y` 确认
3. 从备份恢复上一版代码和数据库，并重启

> 回滚基于部署时的备份，不依赖 Git。

### 4. 从服务器恢复（本地改崩时）

1. 双击 **从服务器恢复.bat**
2. 输入服务器密码
3. 生产包会下载到项目同级目录的 `guaji-从服务器恢复` 文件夹

> 生产环境仅含 dist（编译后 JS）、node_modules、client，不含 TypeScript 源码。若需完整源码，请用 Git 恢复。

---

## 三、脚本目录

实际脚本在 `scripts/` 下：

- `scripts/first-deploy.sh`：首次部署（仅环境）
- `scripts/upload-deploy.ps1`：上传部署（构建 + 打包 + 上传）
- `scripts/deploy-remote.sh`：服务器端部署逻辑（备份、解压、重启）
- `scripts/rollback.sh`：回滚逻辑

**从服务器恢复** 无需 scripts。

---

## 四、生产环境规范

| 规范 | 说明 |
|------|------|
| **仅部署生产包** | 生产服务器不克隆 Git，只接收 dist、node_modules、client、package.json、init-mongodb.js |
| **无源码与测试** | 不包含 src/、scripts/test-*.js、*.test.ts、docs/ 等 |
| **仅运行时依赖** | node_modules 经 `npm prune --production` 精简，不含 Jest、ESLint、TypeScript |
| **部署前自动备份** | 每次部署前备份当前版本到 `/opt/guaji/backups/`，保留最近 5 份 |

## 五、一账号一 IP 防作弊

- **登录**：首次登录记录 IP，后续仅允许同 IP 登录
- **重复登录**：同一账号在新设备/浏览器登录时，旧连接会被踢下线
- **解绑 IP**：用户换网络后无法登录时，管理员在 GM 工具「玩家管理」→「解绑IP」中操作
- **Nginx**：若游戏在 Nginx 后，需配置 `proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;` 以正确获取客户端 IP

## 六、修改服务器地址

若更换服务器，需修改以下文件中的 `120.26.0.177`：

- `scripts/upload-deploy.ps1`（`$SERVER` 变量）
- 各 `.bat` 文件中的 `root@120.26.0.177`

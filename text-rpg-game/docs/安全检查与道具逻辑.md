# 安全检查与道具逻辑

## 一、防漏洞加固（已实现）

### 1. 参数校验

| 接口 | 校验 | 防漏洞 |
|------|------|--------|
| `POST /bag/add` | **已移除** | 玩家不可直接添加物品，防刷道具 |
| `bagService.addItem` | count 1~9999 整数 | 内部调用（商店/拍卖/GM）二次校验 |
| `POST /shop/buy` | count 必须 1~9999 整数 | 已有 |
| `POST /auction/list` | price 必须 >= 0 | 防负价格刷金币 |
| `POST /auction/buy` | count 必须 >= 1 整数 | 已有 |

### 2. 防漏洞测试（test-security.js）

- bag/add 已移除 → 404
- 购买 count=-1/0 → 拒绝
- 拍卖 price=-100 → 拒绝
- 无效 shop_item_id → 拒绝
- 无 token 敏感操作 → 拒绝

### 3. 全量测试包含防漏洞

`test-all.js` 已加入 `test-security.js` 步骤。

## 二、道具可用性（item_effect）

**背包「使用」依赖 item_effect 表**，无配置则提示「该物品无法直接使用」。

| 类型 | item_id | effect_type | 说明 |
|------|---------|-------------|------|
| 消耗品 | 1, 2 | restore | 小血瓶、小蓝瓶（init 已配置） |
| 扩容袋 | 11 | expand_bag | 增加背包装备容量 |
| 多倍卡 | 101-112 | boost | 经验/金币/掉落/声望倍率 |
| 永久属性果实 | 120-125 | add_stat | 永久加属性 |
| VIP 卡 | 201 | vip | 开通 VIP |
| 技能书 | 需 GM 创建 | learn_skill | 学习技能 |

**GM 新建技能书**：在物品管理创建 type=4 道具，选效果「技能书」并填「同时创建技能」，保存后自动写入 item_effect。

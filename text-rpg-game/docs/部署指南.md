# 部署指南

## 一、CI 状态

项目已有 GitHub Actions CI（`.github/workflows/ci.yml`），每次 push/PR 会：

- 构建 `npm run build`
- 初始化 MongoDB
- 运行单元测试
- 启动服务并执行 API + WebSocket 全量测试

**CI 通过 = 代码可部署**。

---

## 二、部署前检查清单

| 项目 | 说明 |
|------|------|
| ✅ CI 通过 | 确保最新代码的 CI 全部通过 |
| ✅ 环境变量 | 生产环境配置 `JWT_SECRET`、`MONGODB_URI` 等 |
| ✅ MongoDB | 生产 MongoDB 已就绪（自建或 Atlas） |
| ✅ 数据库初始化 | 首次部署需执行 `init-mongodb.js` |

---

## 三、环境变量（生产必配）

在部署平台（如 Vercel、Railway、自建服务器）配置：

| 变量 | 必填 | 说明 |
|------|------|------|
| `JWT_SECRET` | ✅ | 强随机字符串，生产必须修改 |
| `MONGODB_URI` | ✅ | MongoDB 连接串（如 `mongodb+srv://...`） |
| `MONGODB_DATABASE` | 否 | 数据库名，默认 `turn-based-game` |
| `PORT` | 否 | HTTP 端口，默认 3000 |
| `WS_PORT` | 否 | WebSocket 端口，默认 3001 |
| `WS_URL` | 否 | 若 WS 经反向代理，设完整 URL（如 `wss://xxx.com/ws`） |

---

## 四、部署步骤

### 1. 首次部署：初始化数据库

```bash
cd text-rpg-game/server
# 设置生产 MongoDB 连接
export MONGODB_URI="mongodb+srv://user:pass@cluster.mongodb.net"
export MONGODB_DATABASE="turn-based-game"
node init-mongodb.js
```

### 2. 构建并启动

```bash
cd text-rpg-game/server
npm ci
npm run build
npm start
```

或使用 PM2：

```bash
pm2 start dist/app.js --name text-rpg
```

### 3. 反向代理（Nginx 示例）

若需 HTTPS 和统一端口：

```nginx
# HTTP API
location /api {
  proxy_pass http://127.0.0.1:3000;
  proxy_http_version 1.1;
  proxy_set_header Host $host;
  proxy_set_header X-Real-IP $remote_addr;
}

# WebSocket
location /ws {
  proxy_pass http://127.0.0.1:3001;
  proxy_http_version 1.1;
  proxy_set_header Upgrade $http_upgrade;
  proxy_set_header Connection "upgrade";
  proxy_set_header Host $host;
}
```

然后设置 `WS_URL=wss://你的域名/ws`，前端会通过 `/api/config/client` 获取该地址。

## 五、本地开发（首次运行）

```bash
# 安装依赖
cd text-rpg-game && npm install
cd server && npm ci

# 安装 Playwright 浏览器（用于 E2E）
cd .. && npx playwright install chromium
```

## 六、CD 部署建议

- **Vercel / Netlify**：本项目为 Node 后端 + 静态前端，需支持 Node 运行时；Vercel 可用 Serverless 部署 `server` 目录。
- **Railway / Render / Fly.io**：适合 Node 后端，配置环境变量后直接部署。
- **自建 VPS**：用 GitHub Actions 或 Jenkins 拉代码 → `npm ci && npm run build` → 重启 PM2。

**原则**：CI 通过 + 环境变量正确 + 数据库已初始化 = 可放心部署。

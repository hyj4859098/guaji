# 安全加固说明

> 上线前安全措施汇总，供部署与审计参考。

---

## 一、认证与密码

| 措施 | 说明 |
|------|------|
| bcrypt 哈希 | 密码使用 bcrypt 存储，不再明文 |
| 明文迁移 | 老用户首次登录时自动迁移为哈希 |
| JWT 算法固定 | 指定 `algorithm: 'HS256'`，防算法混淆 |
| 输入校验 | 用户名 2-32 位、密码 6-128 位、角色名 1-32 位 |

---

## 二、限流

| 接口 | 限制 |
|------|------|
| `/api/user/login` | 10 次 / 15 分钟（同 IP） |
| `/api/user/register` | 同上（与 login 共享） |
| `/api/admin/login` | 同上 |

---

## 三、输入校验与注入防护

| 场景 | 校验 |
|------|------|
| 用户名 | `sanitizeUsername`：字母数字中文下划线 |
| 密码 | `sanitizePassword`：长度 6-128 |
| 角色名 | `sanitizeName`：字母数字中文下划线空格连字符，禁止 `$ . { }` 等 NoSQL 注入字符 |

---

## 四、业务逻辑安全

| 接口 | 防护 |
|------|------|
| 背包 | `bag/add` 已移除，防刷道具；`bag/update` 仅允许 `count`，且 1-9999 |
| 商店 | `shop/buy` 数量 1-9999 校验 |
| 拍卖 | `auction/list` 价格 ≥ 0；`auction/buy` 数量 1-9999 |
| 玩家 | `player/get`、`player/update`、`player/delete` 校验 uid 归属；`player/update` 仅允许 `name`、`auto_battle_config` |

---

## 五、越权防护

- 背包：`delete`、`update`、`wearItem` 校验 uid
- 玩家：`get`、`update`、`delete` 校验角色归属
- 装备：穿戴时校验背包物品归属

---

## 六、防漏洞测试

`node scripts/test-security.js` 覆盖：

- bag/add 已移除
- 商店/拍卖异常数量、负价格
- 无效 ID
- 角色名 NoSQL 注入
- 玩家越权（获取/更新他人角色）
- 无 token 鉴权
- 登录限流 429

---

## 七、部署建议

1. **环境变量**：生产环境使用 `.env`，不提交敏感配置
2. **JWT 密钥**：`JWT_SECRET` 使用强随机字符串
3. **MongoDB**：启用认证，限制网络访问
4. **反向代理**：生产环境使用 Nginx 等，配置 HTTPS、限流
5. **npm audit**：定期执行 `npm audit` 检查依赖漏洞（若使用官方 registry）

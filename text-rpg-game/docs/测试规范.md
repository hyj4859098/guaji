## 一、测试分层与运行方式

| 层级 | 工具 | 命令 | 前置 |
|------|------|------|------|
| 单元测试 | Jest | `cd server && npm run test` | 无 |
| 集成测试 | Jest (MongoDB Memory) | 同上（与单元同跑） | 无 |
| 覆盖率 | Jest | `cd server && npm run test:coverage` | 无 |
| 数据库验证 | verify-init-mongodb | `cd server && npm run test:verify-db` | MongoDB |
| 全量 API/WS/安全 | test-all.js | `node scripts/test-all.js` | MongoDB + 服务已启动 |
| E2E | Playwright | `npx playwright test` | 服务已启动 |
| 上线前全量 | pre-release-check | `npm run pre-release` | MongoDB |

**CI**：`.github/workflows/ci.yml` 在 push/PR 到 main/master/develop 时执行 Lint → Jest（含覆盖率）→ init → verify → 启动服务 → test-all → E2E。

## 二、上线必测清单（按优先级）

| 优先级 | 场景 | 影响 | 验证方式 |
|--------|------|------|----------|
| P0 | 登录/注册/创建角色 | 无法进入游戏 | E2E `auth.spec.js` |
| P0 | 战斗开始/结束、扣血/蓝条 | 核心玩法不可用 | E2E `gameplay.spec.js`、Jest `battle.service` |
| P0 | 背包列表、使用消耗品、穿戴装备 | 无法使用道具 | E2E `gameplay.spec.js`、Jest `bag.service` |
| P0 | 商店购买 | 无法获得物品 | E2E `gameplay.spec.js`、Jest `shop.service` |
| P1 | 拍卖上架/购买 | 交易异常 | Jest `auction.service`、`real-api` |
| P1 | 装备强化/祝福 | 成长线异常 | Jest `equip_upgrade`、`equip_blessing` |
| P1 | 离线挂机 | VIP 体验受损 | Jest `offline-battle.service` |
| P1 | 防刷/越权/注入 | 经济崩溃、作弊 | Jest `security.integration`、`test-security.js` |
| P2 | GM 管理、WebSocket 推送 | 运营/实时体验 | Jest `admin`、`websocket` |

**建议**：每次上线前跑 `npm run pre-release`（全自动，含 E2E 核心玩法），P0 出问题则阻塞发布。**无需人工冒烟**。

## 三、边界测试清单

| 系统 | 边界场景 | 测试位置 |
|------|----------|----------|
| 战斗 | 暴击伤害 ≥ base+1、命中率 0/100、元素克制 | `battle/core.test.ts` |
| 战斗 | 玩家 HP=0 死亡、MP 不足无法放技能 | `battle.service`、`runner` |
| 商店 | count=0/-1、金币不足、商品不存在 | `shop.service`、`security.integration` |
| 拍卖 | price<0、count=0/-1、无效 auction_id | `auction.service`、`security.integration` |
| 背包 | count 1~9999、装备满、物品不存在 | `bag.service` |
| 玩家 | 角色名 NoSQL 注入、更新他人角色 | `security.integration`、`input-sanitize` |
| 强化/祝福 | 材料不足、装备不存在、等级不足 | `equip_upgrade`、`equip_blessing` |
| 离线挂机 | maxSeconds<=0、怪物不存在、VIP 过期 | `offline-battle.service` |

**待补充**：更多边界用例可逐步加入对应 `*.integration.test.ts`。

## 四、防作弊测试清单

| 类型 | 场景 | 验证 |
|------|------|------|
| 防刷道具 | `POST /bag/add` 已移除 → 404 | `security.integration`、`test-security.js` |
| 防刷金 | 拍卖 price<0 拒绝 | 同上 |
| 防异常数量 | 商店/拍卖 count≤0 拒绝 | 同上 |
| 防越权 | 无 token 敏感接口拒绝 | 同上 |
| 防越权 | 更新他人角色（gold 等）拒绝 | 同上 |
| 防注入 | 角色名 `{ "$gt": "" }` 拒绝 | 同上 |
| 限流 | 连续错误登录 11 次 → 429 | `test-security.js`（需真实服务） |
| 无效 ID | shop_item_id/auction_id 不存在拒绝 | 同上 |

**Jest 集成**：`security.integration.test.ts` 覆盖上述大部分，计入覆盖率。  
**脚本**：`node scripts/test-security.js` 需服务已启动，覆盖限流等需真实中间件的场景。

**分模块运行**：

| 命令 | 用例数 | 耗时 |
|------|-------|------|
| `npm run e2e:auth` | 4 | ~15s |
| `npm run e2e:gameplay` | 22 | ~8min |
| `npm run e2e:multi` | 3 | ~5min |
| `npm run e2e` | 29 | ~20min |

**推荐修复流程**（省资源）：

1. 跑一次全量：`npm run e2e`
2. 修失败用例后，只重跑失败：`npm run e2e:retry`（`--last-failed`，4 workers）
3. 全部修完后再跑一次全量确认

## 六、覆盖率要求

| 指标 | 门槛 | 说明 |
|------|------|------|
| statements | 89% | `jest.config.js` |
| branches | 72% | 同上 |
| functions | 94% | 同上 |
| lines | 91% | 同上 |

- **统计范围**：`src/**/*.ts`，排除 `app.ts`、`config/db.ts`、`__test-utils__`、`*.test.ts`。
- **单元 + 集成**：两套 project 共用同一门槛，任一不达标则 `npm run test:coverage` 失败。
- **当前用例数**：约 880+（单元 + 集成），随新增测试递增。

# 文字回合制挂机游戏 - 完整测试方案

## 一、测试层级

| 层级 | 工具/脚本 | 运行条件 | 覆盖范围 |
|------|------------|----------|----------|
| L1 单元测试 | Jest | 无依赖 | 工具函数、战斗核心逻辑 |
| L2 数据库验证 | verify-init-mongodb.js | MongoDB | 集合、配置、功能性道具 |
| L3 API 集成 | test-gm-apis.js, test-player-apis.js | MongoDB + 服务器 | GM 管理端、玩家端 HTTP API |
| L4 WebSocket | test-websocket.js | MongoDB + 服务器 | 连接、心跳、聊天、Boss 订阅 |
| L5 业务流程 | test-player-apis.js（扩展） | MongoDB + 服务器 | 战斗、商店、背包、装备 |
| L6 全量测试 | test-all.js | MongoDB + 服务器 | 上述全部串联 |

## 三、执行命令

```bash
cd text-rpg-game/server

# 1. 单元测试（无需服务）
npm run test

# 2. 数据库验证（需 MongoDB）
npm run test:verify-db

# 3. 启动服务器后执行
npm run dev   # 另开终端保持运行

# 4. 全量测试（单元 + DB + GM API + 玩家 API + WebSocket）
npm run test:full
```

`test:full` 会按顺序执行：
1. Jest 单元测试
2. init-mongodb 验证
3. GM API 测试
4. 玩家 API 测试（含注册、战斗、背包、自动战斗等）
5. WebSocket 测试（连接、心跳、聊天、Boss 订阅）

# 前端构建迁移方案：从 Script Tags 到 Vite + ES Modules

## 现状

前端使用 `<script>` 标签按顺序加载所有 JS 文件，依赖全局变量（`State`, `API`, `WS`, `Helper` 等）。

**问题**：
- 无 Tree Shaking，所有代码都会加载
- 无代码压缩/minify
- 隐式依赖关系，加载顺序错误会导致 ReferenceError
- 无法使用 npm 前端库
- 无 HMR，修改前端需手动刷新

## 迁移方案

### 阶段 1：准备（当前已完成）

- [x] 提取公共组件到 `shared-ui.js`
- [x] 引入 CSS 变量（`variables.css`）
- [x] 统一 token 来源

### 阶段 2：Vite 引入（渐进式）

1. 在 `text-rpg-game/client` 下初始化 Vite：

```bash
cd text-rpg-game
npm init -y
npm install -D vite
```

2. 创建 `vite.config.js`：

```js
import { defineConfig } from 'vite';
export default defineConfig({
  root: 'client',
  build: { outDir: '../dist-client' },
  server: { proxy: { '/api': 'http://localhost:3000', '/ws': { target: 'ws://localhost:3001', ws: true } } }
});
```

3. 将 `index.html` 改为 Vite 入口：
   - 移除所有 `<script>` 标签
   - 添加 `<script type="module" src="/js/main.js"></script>`

4. 将每个全局对象改为 ES Module 导出：
   - `state.js` → `export const State = { ... }`
   - `api.js` → `import { State } from './state.js'; export const API = { ... }`
   - 每个 page 文件 → `import { API, State, ... } from '../core/index.js'`

### 阶段 3：逐文件迁移

建议按依赖顺序迁移：
1. `core/state.js` → ES Module
2. `core/api.js` → ES Module（依赖 state）
3. `core/ws.js` → ES Module（依赖 state）
4. `core/helper.js`, `core/ui.js` 等
5. `pages/*` 逐个迁移

每迁移一个文件，确保其他文件仍能通过全局变量访问（过渡期同时导出到 window）。

### 阶段 4：清理

- 移除 `window.XXX = XXX` 兼容代码
- 启用 Vite 生产构建
- 服务器 `express.static` 指向 `dist-client`
- 更新部署脚本包含前端构建步骤

## 风险

- 迁移期间需要维护两套加载方式
- E2E 测试可能需要调整
- GM 工具使用独立的 ES Module 系统，不受影响

## 预估工作量

- 阶段 2：2-3 小时
- 阶段 3：每个文件 15-30 分钟，约 20 个文件 = 1-2 天
- 阶段 4：2-3 小时

# 系统架构文档

## 1. 项目概述

这是一个基于 Node.js + TypeScript + Express + MySQL + WebSocket 的文本RPG游戏后端系统。系统采用分层架构，将业务逻辑与数据访问分离，确保代码的可维护性和可扩展性。

## 2. 目录结构

```
/src
  /api            # 接口层，处理HTTP请求
  /service        # 业务逻辑层，实现核心功能
  /model          # 数据访问层，处理数据库操作
  /types          # 类型定义和枚举
  /middleware     # 中间件，处理鉴权、错误、日志等
  /event          # WebSocket 推送事件
  /utils          # 工具函数和工具类
  /config         # 配置文件
  app.ts          # 项目入口
```

## 3. 核心模块

### 3.1 玩家模块（Player）

- **功能**：处理玩家信息的增删改查、经验值管理、金币管理和声望管理
- **文件**：
  - `src/service/player.service.ts` - 玩家业务逻辑
  - `src/model/player.model.ts` - 玩家数据访问
  - `src/api/player.ts` - 玩家API接口
- **依赖**：LevelExpService（用于计算等级经验）

### 3.2 装备模块（Equip）

- **功能**：处理装备的增删改查、穿戴和卸下装备
- **文件**：
  - `src/service/equip.service.ts` - 装备业务逻辑
  - `src/model/equip.model.ts` - 装备数据访问
  - `src/api/equip.ts` - 装备API接口
- **依赖**：EquipEffectUtil（用于处理装备效果）

### 3.3 战斗模块（Battle）

- **功能**：处理玩家与怪物的战斗逻辑，支持实时战斗、自动战斗和分步战斗
- **文件**：
  - `src/service/battle.service.ts` - 战斗业务逻辑
  - `src/model/battle.model.ts` - 战斗数据访问
  - `src/api/battle.ts` - 战斗API接口
- **依赖**：PlayerService, MonsterService, BattleStateMachine

### 3.4 背包模块（Bag）

- **功能**：处理玩家背包物品的管理
- **文件**：
  - `src/service/bag.service.ts` - 背包业务逻辑
  - `src/model/bag.model.ts` - 背包数据访问
  - `src/api/bag.ts` - 背包API接口

### 3.5 WebSocket事件管理

- **功能**：处理实时通信，推送战斗事件、玩家状态更新等
- **文件**：
  - `src/event/ws-manager.ts` - WebSocket连接管理

## 4. 核心流程

### 4.1 战斗流程

1. 玩家发起战斗请求，指定怪物ID和战斗模式（自动/手动）
2. 战斗服务创建战斗状态机，初始化战斗状态
3. 执行战斗逻辑，计算伤害、命中、闪避等
4. 推送战斗事件到前端，包括战斗开始、回合开始、攻击、战斗结束等
5. 处理战斗结算，包括经验值、金币、声望的奖励
6. 推送奖励信息到前端
7. 清理战斗状态

### 4.2 装备穿戴流程

1. 玩家发起穿戴装备请求
2. 装备服务检查装备是否属于该玩家
3. 检查装备位置是否已有装备，如有则卸下并移除装备效果
4. 穿戴新装备，应用装备效果
5. 更新数据库记录
6. 推送装备状态更新到前端

## 5. 技术选型

- **运行环境**：Node.js
- **开发语言**：TypeScript
- **后端框架**：Express
- **数据库**：MySQL
- **实时通信**：WebSocket
- **登录鉴权**：JWT
- **依赖**：express、ws、jsonwebtoken、mysql2

## 6. 设计原则

1. **分层架构**：严格分离API层、服务层、数据访问层，确保职责清晰
2. **类型安全**：使用TypeScript确保类型安全，减少运行时错误
3. **可扩展性**：模块化设计，便于添加新功能
4. **错误处理**：统一的错误处理机制，确保系统稳定性
5. **日志系统**：完善的日志记录，便于问题排查
6. **安全性**：JWT鉴权，防止未授权访问

## 8. 部署建议

- **环境**：Node.js 14+，MySQL 5.7+
- **配置**：根据服务器性能调整端口、数据库连接池等配置
- **监控**：建议使用PM2进行进程管理和监控
- **备份**：定期备份数据库，确保数据安全

## 9. 开发规范

- **代码风格**：遵循TypeScript标准风格，使用ESLint进行代码检查
- **命名规范**：
  - 字段命名：小写+下划线（如：player_hp）
  - 函数命名：小驼峰（如：getPlayerInfo）
  - 文件命名：功能.类型.ts（如：player.service.ts）
- **注释规范**：使用JSDoc注释，确保代码可维护性
- **提交规范**：遵循Git提交规范，提交信息清晰明了

## 10. 系统性能优化

- **数据库优化**：合理设计数据库表结构，添加索引
- **缓存机制**：对频繁访问的数据使用缓存
- **异步处理**：使用异步/await处理IO操作，避免阻塞
- **WebSocket优化**：合理管理WebSocket连接，避免内存泄漏
- **战斗系统优化**：减少setTimeout使用，考虑使用事件循环或定时器

## 11. 安全考虑

- **输入验证**：对所有用户输入进行验证，防止SQL注入等攻击
- **密码加密**：使用bcrypt等算法对密码进行加密存储
- **JWT安全**：设置合理的JWT过期时间，使用安全的密钥
- **CORS配置**：合理配置CORS，防止跨域攻击
- **日志安全**：避免在日志中记录敏感信息

## 12. 总结

本系统采用现代化的技术栈和架构设计，确保了系统的可维护性、可扩展性和安全性。通过分层架构和模块化设计，使得系统易于理解和维护，同时为未来的功能扩展预留了空间。
